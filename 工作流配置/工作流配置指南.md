# Dify 工作流配置指南

> 在 Dify Cloud 中创建 Chatflow 类型应用，按以下步骤配置
> 已通过真实测试验证

---

## 一、创建应用

1. 登录 Dify Cloud → 创建应用 → 选择「Chatflow」
2. 应用名称：`电商智能客服助手`
3. 描述：基于知识库+意图路由的智能客服，支持智能问答、风险识别、工单自动生成

---

## 二、知识库配置

### 上传文档
1. 进入「知识库」→「创建知识库」
2. 名称：`电商售后知识库`
3. 上传以下**5个独立文件**（拆分为小文件，提升关键词检索命中率）：
   - `T恤-商品详情.md`（材质面料成分尺码颜色洗涤）
   - `牛仔裤-商品详情.md`（材质面料成分弹力版型尺码）
   - `羽绒服-商品详情.md`（材质填充物鸭绒保暖洗涤尺码）
   - `卫衣-商品详情.md`（材质毛圈加绒起球变形版型）
   - `售后政策知识库.md`（退换货/退款/物流/订单）
   - `通用FAQ-物流支付售后.md`（物流支付发票优惠券）
4. 上传后等待 Embedding 处理完成

### 切片设置
```
分段模式：自动分段
分段最大长度：800 tokens（调大以保证"材质"关键词不被切碎）
分段重叠长度：80 tokens
索引方式：高质量模式
```

### 检索设置（知识检索节点中配置）
```
检索模式：混合检索
语义权重：0.7
关键词权重：0.3
Top K：3
Score 阈值：0.4
```

### 关键优化：关键词铺底策略
每个知识库文件的标题和首段都包含高频关键词（材质 面料 成分 棉...），确保关键词检索一定能命中。搜索"材质"会命中所有文件。

## 三、工作流节点配置

### 完整节点拓扑（含安全防护 + 埋点）

```
[开始]
  ↓
[Code: 输入脱敏]        安全第一层：手机号/身份证/订单号脱敏
  ↓
[LLM: 安全检测]         安全第二层：Prompt注入/政治敏感/色情暴力检测
  ↓                      → safe=false → [直接回复: 安全拦截]
[LLM: 意图分类]         DeepSeek-V4, Temp=0.1, MaxToken=512
  ↓
[Code: 解析意图]         清理<think>标签 + JSON解析
  ↓
[Code: 意图埋点]         埋点事件：intent_classify
  ↓
[条件分支: 路由判断]
  ├─ CASE1: intent是"投诉" OR intent是"退换货"
  │    ├─ [LLM: 工单生成]
  │    ├─ [Code: 工单创建]  → 构造工单对象 + 飞书字段
  │    ├─ [Code: 工单埋点]  → 埋点事件：ticket_create
  │    ├─ [Code: 输出审核]  → 安全第三层
  │    └─ [直接回复: 工单确认]
  ├─ CASE2: confidence < 0.6
  │    ├─ [Code: BadCase采集] → 埋点事件：transfer_human
  │    └─ [直接回复: 转人工]
  └─ ELSE
       └─ [知识检索]         混合检索, Score≥0.4
            ├─ [Code: 检索埋点] → 埋点事件：knowledge_search
            └─ [条件分支: 检索判断]
                 ├─ result不为空 → [LLM: 生成回答]
                 │    ├─ [Code: 回答埋点] → 埋点事件：answer_generate
                 │    ├─ [Code: 输出审核] → 安全第三层
                 │    └─ [直接回复]
                 └─ ELSE → [Code: BadCase采集] → [直接回复: 转人工]
```

## 四、各节点详细配置

### 节点1：开始节点
| 变量名 | 类型 | 必填 |
|--------|------|------|
| history | Text | 否 |

### 节点2：LLM - 意图分类
| 配置项 | 值 | 说明 |
|--------|-----|------|
| 模型 | DeepSeek-V4 | 性价比最高 |
| 温度 | 0.1 | 低温度保证分类稳定 |
| Max Token | 512 | 需足够大容纳<think>+JSON |
| 结构化输出 | 关闭 | DeepSeek 不稳定 |
| SYSTEM | 见 Prompt模板.md V2.0 | 含关键判断规则 |
| USER | {{#userinput.query#}} | 用变量选择器选 |


**踩坑记录：**
- MaxToken=256 时，`<think>` 标签消耗完 Token，JSON 截断 → finish_reason=length
- MaxToken=512 解决
- USER 提示词手写变量名 → 报"无效变量"
- 用变量选择器选

### 节点3：Code - 解析意图
| 配置项 | 值 |
|--------|-----|
| 输入变量 | intent_result = {{节点2.text}} |
| 代码 | 见 Prompt模板.md 节点2 |

**输出变量：**
| 变量名 | 类型 | 示例 |
|--------|------|------|
| intent | String | 售前咨询 |
| confidence | Number | 0.95 |
| reason | String | 询问商品材质 |

**踩坑记录：**
- 初始版本没有 `<think>` 清理 → JSON 解析失败，confidence=0
- 4层容错处理（think标签→代码块→正则提取→直接解析）

### 节点4：条件分支 - 路由
| 分支 | 条件 | 跳转 |
|------|------|------|
| CASE 1 | {{节点3.intent}} **是** "投诉" OR {{节点3.intent}} **是** "退换货" | 工单生成 |
| CASE 2 | {{节点3.confidence}} **<** 0.6 | 转人工提示 |
| ELSE | 默认 | 知识检索 |

**注意：** Dify 条件分支用"是"不是"等于"

### 节点5：知识检索
| 配置项 | 值 |
|--------|-----|
| 知识库 | 电商售后知识库 |
| 查询内容 | {{#userinput.query#}} |
| 检索模式 | 混合检索 |
| 语义权重 | 0.7 |
| 关键词权重 | 0.3 |
| Top K | 3 |
| Score 阈值 | 0.4 |

**踩坑记录：**
- Score阈值=0.6 → 实际召回分数0.53~0.56，全部被过滤，result=[]
- Score阈值=0.4 解决

### 节点6：条件分支 - 检索判断
| 分支 | 条件 | 跳转 |
|------|------|------|
| CASE 1 | {{知识检索.result}} **不为空** | LLM 生成回答 |
| ELSE | 默认 | 转人工提示 |

### 节点7：LLM - 工单生成
| 配置项 | 值 |
|--------|-----|
| 模型 | DeepSeek-V4 |
| 温度 | 0.1 |
| Max Token | 256 |
| SYSTEM | 见 Prompt模板.md 节点4 |
| USER | 用户问题：{{#userinput.query#}} 意图：{{节点3.intent}} |

### 节点8：LLM - 生成回答
| 配置项 | 值 |
|--------|-----|
| 模型 | DeepSeek-V4 |
| 温度 | 0.3 |
| Max Token | 512 |
| 上下文 | 开启，关联知识检索节点 |
| SYSTEM | 见 Prompt模板.md 节点3 |

### 回复节点文案

**转人工提示：**
```
亲，我不太确定您的具体需求呢~ 为了更好地帮您解决问题，建议转接人工客服哦！

工作时间：每天 9:00 - 22:00
输入「转人工」可立即接入
非工作时间会优先在次日处理
```

**工单确认：**
```
亲，您反馈的问题已帮您生成工单，我们将尽快处理！

工单详情：{{工单LLM.text}}

如需加急处理，建议联系人工客服：
输入「转人工」立即接入
工作时间：每天 9:00 - 22:00
```
