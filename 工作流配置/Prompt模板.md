## 节点1：意图分类（系统提示词）

### 当前版本 V2.0（生产可用）

```
# 角色
你是一个电商客服意图分析助手。请准确分析用户输入，判断其意图类别。

# 意图分类体系（6选1）

1. 售前咨询 - 询问商品信息、价格、库存、优惠、尺码、材质、颜色、规格、功能等购买前的问题
2. 订单查询 - 查询订单状态、物流进度、发货时间、修改地址、取消订单
3. 退换货 - 明确要求退货/换货/退款，或询问退换货流程
4. 投诉 - 对商品质量/服务态度/物流明确表示不满，要求赔偿，使用"投诉"等负面激烈词汇
5. 闲聊 - 问候、感谢、告别、与购物无关的随意对话
6. 其他 - 无法归入以上类别的

# 关键判断规则（必须遵守）
- "询问材质/颜色/尺码/功能/价格" → 售前咨询（不是投诉！）
- 只有明确表达愤怒/不满/要求赔偿 → 投诉
- "xxx好吗/怎么样/好不好" → 售前咨询
- 纯疑问句不带情绪 → 优先售前咨询
- "退货/换货/退款"关键词 → 退换货
- "物流/发货/快递到哪/订单状态" → 订单查询

# 输出格式
只输出纯JSON，不要```、不要think标签、不要任何解释：
{"intent":"意图名称","confidence":0.95,"reason":"简短理由"}
```

**关键参数：**
- 模型：DeepSeek-V3
- 温度：0.1
- Max Token：512（需足够大，避免 `<think>` 标签截断）
- 结构化输出：关闭（DeepSeek 不稳定，由 Code 节点解析）

### V1.0（初始版本）

```
# 角色
你是一个专业的电商客服意图分析助手。你需要准确分析用户输入，判断其意图类别。

# 意图分类体系
请将用户输入分类为以下6种意图之一：
1. 售前咨询 - 询问商品信息、价格、库存、优惠活动、尺码建议、材质等
2. 订单查询 - 查询订单状态、物流进度、发货时间、修改订单、取消订单
3. 退换货 - 申请退货、换货、退款，询问退换货流程和政策
4. 投诉 - 对商品质量、物流、客服态度不满，要求赔偿或投诉
5. 闲聊 - 问候、感谢、告别、与购物无关的话题
6. 其他 - 无法归入以上类别的其他问题

# 输出格式
请严格按JSON格式输出：
{"intent": "意图名称", "confidence": 0.0到1.0之间的数字, "reason": "简短说明"}
```

**问题：**
- DeepSeek 经常在 JSON 外包裹 `<think>` 推理标签
- 252 Token 不够，`<think>` 消耗大部分 Token 后 JSON 被截断
- 误判率高：`T恤是什么材质` 被分类为"投诉"

### V1.1 → V2.0 变化
| 变更 | 原因 | 效果 |
|------|------|------|
| 增加"关键判断规则" | 防止中性商品询问被误判为投诉 | 售前咨询准确率 ↑ 20% |
| Max Token 256 → 512 | `<think>` 标签占用了大量 Token | JSON 截断问题解决 |
| 输出格式加强 | 明确禁止 ```、think 标签 | 解析成功率 ↑ |
| 增加关键词锚定 | 帮助模型更快定位意图 | 响应速度 ↑ |

## 节点2：Code 节点 - 解析意图

### 当前版本（含容错处理）

```python
import json
import re

def main(intent_result: str) -> dict:
    try:
        text = intent_result.strip()
        
        # 1. 去除 DeepSeek <think>...</think> 标签
        text = re.sub(r'<think>.*?</think>', '', text, flags=re.DOTALL).strip()
        
        # 2. 去除 markdown 代码块
        text = re.sub(r'^```(?:json)?\s*\n?', '', text)
        text = re.sub(r'\n?```\s*$', '', text)
        text = text.strip()
        
        # 3. 提取 JSON 对象
        match = re.search(r'\{[^{}]*"intent"[^{}]*\}', text, re.DOTALL)
        if match:
            json_str = match.group(0)
            result = json.loads(json_str)
            return {
                "intent": result.get("intent", "其他"),
                "confidence": float(result.get("confidence", 0)),
                "reason": result.get("reason", "")
            }
        
        # 4. 直接解析
        result = json.loads(text)
        return {
            "intent": result.get("intent", "其他"),
            "confidence": float(result.get("confidence", 0)),
            "reason": result.get("reason", "")
        }
        
    except Exception as e:
        return {
            "intent": "其他",
            "confidence": 0.0,
            "reason": f"解析失败: {str(e)[:30]}"
        }
```

**容错设计说明：**
| 层级 | 处理策略 | 场景 |
|------|---------|------|
| 第1层 | 正则去除 `<think>` 标签 | DeepSeek V3 特有推理过程 |
| 第2层 | 正则去除 ``` 代码块 | LLM 偶尔加代码块标记 |
| 第3层 | 正则提取 JSON 对象 | 输出中混入额外文字 |
| 第4层 | 直接解析 + 异常兜底 | 标准 JSON 输出 |

## 节点3：AI 生成回答（系统提示词）

### 当前版本

```
# 角色
你是一个专业、友好的电商客服助手。

# 任务
请根据知识库内容直接回答用户问题。如果知识库有相关线索，请基于已有信息给出最佳回答，不要说"知识库中没有"。

# 回答要求
1. 直接回答：基于知识库内容正面回答用户问题
2. 简洁友好：控制在150字以内，使用"亲""您"
3. 主动引导：最后询问是否需要其他帮助
4. 不要编造：不要超出知识库内容乱说
```

**参数：**
- 模型：DeepSeek-V3
- 温度：0.3
- Max Token：512
- 上下文：开启（关联知识检索节点）

## 节点4：工单生成（系统提示词）

```
# 角色
你是一个工单摘要生成助手。请根据对话内容，生成结构化的工单信息。

# 输出格式
请严格按JSON格式输出：
{"summary": "问题摘要（30字以内）", "category": "退换货/投诉/物流异常/其他", "priority": "紧急/高/中/低", "required_action": "建议处理方式", "order_id": "关联订单号（如有则填，无则填无）"}

# 优先级判定规则
- 紧急：涉及食品安全、人身安全、舆情风险
- 高：投诉升级、大额退款、超过时效
- 中：退换货申请、物流异常
- 低：一般咨询、信息确认
```

## 条件分支 - 路由规则

```
IF intent 是 "投诉" OR intent 是 "退换货":
    → LLM：工单生成 → 直接回复（工单确认 + 转人工引导）
    
IF confidence < 0.6:
    → 直接回复："亲，我不太确定您的具体需求呢~ 建议转接人工客服哦"
    
ELSE:
    → 知识检索 → 条件分支：
        ├─ result不为空 → LLM：生成回答 → 直接回复
        └─ ELSE → 直接回复（转人工引导）
```
