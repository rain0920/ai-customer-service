# 用户分层与场景定义

> 从"面向中小电商商家"的一句话定位，到可落地的用户分层和场景设计

---

## 一、用户分层

### 1.1 分层维度

| 维度 | 说明 | 为什么重要 |
|------|------|-----------|
| 日咨询量 | 日均用户消息数 | 决定算力成本和服务等级 |
| 类目 | 服装/美妆/3C/食品/家居 | 不同类目知识库结构完全不同 |
| 平台数量 | 单平台 vs 多平台 | 多平台商家更需要跨平台统一方案 |
| 客服人力 | 0人/1-2人/3人以上 | 决定对自动化的依赖程度 |

### 1.2 四层用户画像

#### L1：微型商家（MVP 首发目标）

| 特征 | 描述 |
|------|------|
| 日咨询量 | < 50 条 |
| 典型类目 | 服装、饰品、手作 |
| 平台 | 1-2 个（淘宝+小红书/抖音小店） |
| 客服人力 | 0-1 人（通常是店主自己） |
| 月营收 | < 10 万 |
| 核心痛点 | 没人值班时丢单；回复慢被差评；不会配置复杂系统 |
| 付费意愿 | 月费 < 100 元 |
| 决策关键词 | **便宜、简单、能跑就行** |

> **为什么先做 L1：** 这是市场空白（店小蜜免费版功能受限，乐言年费 3 万+），且需求最刚性——微型商家客服能力为 0，AI 是唯一解而非替代品。

#### L2：小型商家（V2.0 拓展目标）

| 特征 | 描述 |
|------|------|
| 日咨询量 | 50-200 条 |
| 典型类目 | 服装、美妆、母婴 |
| 平台 | 2-3 个 |
| 客服人力 | 1-2 人 |
| 月营收 | 10-50 万 |
| 核心痛点 | 高峰期人力不足；咨询转化率低；售后退款纠纷多 |
| 付费意愿 | 月费 100-500 元 |
| 决策关键词 | **分流率、转化提升、工单闭环** |

#### L3：中型商家（远期规划）

| 特征 | 描述 |
|------|------|
| 日咨询量 | 200-1000 条 |
| 典型类目 | 全类目 |
| 平台 | 多平台+私域 |
| 客服人力 | 3-10 人 |
| 核心痛点 | 多渠道统一管理；数据安全；质检合规 |
| 付费意愿 | 月费 500-3000 元 |

#### L4：头部商家（不在本产品范围内）

日咨询 > 1000，需要私有化部署、定制开发、专属运维——交给乐言/智齿。

## 二、场景定义

### 2.1 核心场景矩阵

按"用户生命周期 × 咨询意图"两个维度交叉定义场景：

| | 售前（购买决策） | 售中（下单发货） | 售后（退换投诉） |
|---|---|---|---|
| **闲时**（日常） | 场景A：商品咨询 | 场景B：物流查询 | 场景D：退换货处理 |
| **忙时**（大促/上新） | 场景A+：高并发咨询 | 场景B+：催发货 | 场景D+：批量售后 |

### 2.2 场景 A：售前商品咨询（最高频，占比 ~65%）

**典型对话流程：**

```
用户："这件T恤是什么材质？"
  → 意图分类：售前咨询
  → 知识检索：命中 T恤-商品详情
  → AI回答："亲，100%新疆长绒棉，亲肤透气～"

用户："会缩水吗？"（多轮追问）
  → Query改写："这件T恤会缩水吗？"
  → 知识检索：命中缩水率信息
  → AI回答："经过预缩处理，缩水率<3%哦～"

用户："白色有现货吗？M码"
  → AI回答后引导："有现货的，需要帮您下单吗？"
```

**子场景覆盖：**

| 子场景 | 高频问题 | 知识库覆盖要求 |
|--------|---------|--------------|
| 材质咨询 | "什么面料""会起球吗""透吗" | 面料成分、工艺说明 |
| 尺码咨询 | "160穿什么码""偏大还是偏小" | 尺码表+试穿建议 |
| 颜色咨询 | "白色会透吗""实物色差大吗" | 颜色实拍+描述 |
| 价格优惠 | "有优惠吗""能便宜吗" | 活动规则、优惠券 |
| 对比咨询 | "这两款有什么区别" | 商品对比信息 |

**忙时特征（大促期间）：**
- 并发量 3-5 倍于日常
- 用户耐心更低，回复速度要求 < 10 秒
- 常见问题高度集中（"什么时候发货""能优惠吗"）

### 2.3 场景 B：物流/订单查询（占比 ~15%）

**典型对话流程：**

```
用户："我的订单发货了吗？"
  → 意图分类：订单查询
  → 知识检索：命中发货政策
  → AI回答：说明发货时效+引导查看订单详情
```

**关键设计决策：**
- 物流查询本质上是**数据查询问题**（需要对接订单系统 API），不是纯文本问答
- MVP 阶段用 FAQ 文本兜底（"通常 24 小时内发货"），V2.0 对接订单 API 实现精准查询

### 2.4 场景 C：退换货处理（占比 ~12%，高风险）

**典型对话流程：**

```
用户："买大了想换小一码"
  → 意图分类：退换货
  → 工单生成 → 工单确认回复
  → 引导用户走退换货流程
```

**风险点：**
- 退换货直接关联金钱，回复错误可能导致资损
- 需要收集关键信息：订单号、退换原因、商品状态（吊牌是否剪掉）
- 情绪敏感，回复语气需格外谨慎

### 2.5 场景 D：投诉（占比 ~5%，最高风险）

**典型对话流程：**

```
用户："你们客服态度太差了，我要投诉！"
  → 意图分类：投诉
  → 工单生成（priority=高/紧急）
  → 安抚话术 + 升级处理引导
```

**处理原则：**
- 绝不自动处理，必须走工单+转人工
- 安抚情绪优先，解决问题其次
- 紧急投诉（舆情风险、食品安全）需即时通知人工

### 2.6 场景 E：闲聊/其他（占比 ~3%）

问候、感谢、告别等。AI 友好回应即可，不需要检索知识库。

## 三、MVP 阶段聚焦策略

### 为什么聚焦服装类目 + L1 商家？

| 决策依据 | 说明 |
|----------|------|
| 类目选择：服装 | ①售前咨询占比最高（65%），AI 替代价值最大；②问题类型集中（材质/尺码/颜色），知识库相对简单；③竞品晓多已验证服饰是 AI 客服最优类目 |
| 用户选择：L1 微型商家 | ①市场空白，无竞品覆盖；②需求刚性（0 客服人力）；③获客成本低（社群/小红书可触达） |
| 平台选择：淘宝+抖音小店 | ①淘宝商家最多；②抖音小店是增长最快的新平台，商家客服能力普遍更弱 |

### MVP 成功指标

| 指标 | 目标值 | 衡量方式 |
|------|--------|---------|
| 意图识别准确率 | ≥ 90% | 测试集 100 条 + 线上抽样 |
| 知识检索命中率 | ≥ 80% | 检索结果非空比例 |
| 转人工率 | ≤ 30% | 实际转人工次数 / 总咨询次数 |
| 用户满意度 | ≥ 4.0/5 | 对话结束后的满意度评分 |
| 单店月成本 | < 50 元 | DeepSeek API 实际消耗 |

## 四、面试叙事模板

> "在项目启动阶段，我没有直接写 Prompt，而是先做了用户分层和场景定义。
>
> 通过竞品分析我发现，店小蜜统治淘宝生态，乐言/晓多服务中大型商家，而**日咨询量 < 50 的微型多平台商家**是一个市场空白——他们客服能力为 0，付费能力弱，头部产品看不上这个市场。
>
> 我选择**服装类目**作为切入点，因为售前咨询占比 65%，且问题集中在材质/尺码/颜色三类——这意味着用较少的规则和知识就能覆盖大部分场景。在 MVP 阶段，我只聚焦 L1 微型商家 + 服装类目，用最少的资源验证产品价值。"
