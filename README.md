# 智能客服助手

> Dify Cloud 智能客服助手

---

## 项目概述

基于 Dify Chatflow + DeepSeek + 知识库（RAG）构建的电商智能客服系统，支持“意图识别 → 风险分级 → 智能回答 / 工单生成 / 转人工”的多级路由策略。

### 核心差异化
- **不是简单问答匹配**：6类意图分类 + 置信度阈值 + 高危意图工单生成
- **不是调个 API**：完整工作流编排
- **不是黑盒**：100条测试用例 + 效果评估体系 + Bad Case 追踪

---

## 技术架构

```
用户输入 → 意图分类(DeepSeek) → Code解析 → 条件路由
                                              ├─ 投诉/退换货 → 工单生成 → 人工引导
                                              ├─ 低置信度(<0.6) → 人工引导
                                              └─ 正常意图 → 知识检索(RAG) → AI回答
```

| 组件 | 选型 | 原因 |
|------|------|------|
| 工作流平台 | Dify Cloud (Chatflow) | 可视化编排，运营可维护 |
| LLM | DeepSeek-V3 | 性价比最高，中文能力强 |
| Embedding | BAAI/bge-large-zh-v1.5 (SiliconFlow) | 中文语义检索效果最佳 |
| 知识库 | Dify 内置 RAG | 混合检索，免开发 |
| 测试框架 | Dify 批量测试 + 人工评测 | 量化效果 |

## 意图分类体系

| 意图 | 路由策略 | 示例 |
|------|----------|------|
| 售前咨询 | 知识检索 → AI回答 | "T恤是什么材质" |
| 订单查询 | 知识检索 → AI回答 | "快递到哪了" |
| 退换货 | 工单生成 + 人工引导 | "买大了想换小一码" |
| 投诉 | 工单生成 + 人工引导 | "客服态度太差" |
| 闲聊 | AI友好回应 | "你好呀" |
| 其他 | 转人工 | 无法归类的问题 |

## 踩坑记录

| 问题 | 现象 | 方案 |
|------|------|------|
| DeepSeek `<think>` 标签污染 | JSON 解析全部失败 | Code节点正则清理 |
| MaxToken 不足 | `<think>` 耗完 Token，JSON 截断 | 256→512 |
| 意图误判 | "T恤是什么材质"→投诉 | Prompt 增加关键判断规则 |
| Score 阈值过高 | 知识检索永远返回空 | 0.6→0.4 |
| Dify 语法差异 | "等于"不存在 | 改用"是" |
| 关键词检索命中率为0 | 搜索"材质"召回为空 | 拆分独立小文件+关键词铺底 |
